let (tx, _) = broadcast::channel::(1024); broadcast คือ class หรือ cargo ที่ทำหน้าที่ ส่งหรือ กระจายข้อมูลไปยัง ผู้รับ : เป็นการระบุประเภทข้อมูล (Generic Type) ว่าท่อนี้จะรับส่งเฉพาะข้อมูลที่เป็น Struct Candle เท่านั้น (1024): คือขนาดของ Buffer (หน่วยความจำสำรอง) หมายความว่าท่อนี้สามารถเก็บข้อมูลที่รอการประมวลผลไว้ได้สูงสุด 1,024 ชุด หากข้อมูลไหลมาเร็วมากจนผู้รับอ่านไม่ทัน ข้อมูลเก่าที่สุดจะถูกลบออกเพื่อรับข้อมูลใหม่ (ป้องกัน Memory รั่วไหล) let (tx, _) ผลลัพธ์จากการสร้าง Channel จะคืนค่ากลับมาเป็น Tuple สองอย่าง: tx (Transmitter): คือ "หัวส่ง" หรือ "ไมโครโฟน" เราจะใช้ตัวแปรนี้ในการ send ข้อมูล Candle เข้าไปในท่อ _ (Receiver): ปกติจุดนี้คือ "หัวรับ" ตัวแรกที่สร้างขึ้นพร้อม Channel แต่เนื่องจากในระบบของเรา เราต้องการสร้างหัวรับใหม่แยกกันในทุก ๆ ครั้งที่มี Browser คนใหม่เชื่อมต่อเข้ามา (Dynamic Subscribers) เราจึงใช้เครื่องหมาย _ เพื่อบอก Rust ว่า "ไม่ต้องเก็บหัวรับตัวแรกนี้ไว้" **** ถ้าไม่มี tx ตัวนี้ เราจะไม่สามารถส่งข้อมูลราคาจากงานพื้นหลัง (Background Task) กลับไปหาหน้าเว็บได้เลยครับ *** ในโปรเจกต์นี้ tx (Transmitter) คือหัวใจสำคัญในการกระจายข้อมูลราคาจาก Deriv ไปยังหน้าจอต่าง ๆ ของผู้ใช้ทุกคน โดยมีการใช้คำสั่งที่เกี่ยวข้องดังนี้ครับ: 1. การสร้างตัวส่งข้อมูล (Initialization)คำสั่ง: let (tx, _) = broadcast::channel::(1024); ใช้เพื่อ: สร้าง "สถานีส่งสัญญาณ" หลักขึ้นมาภายในหน่วยความจำของ Rustอธิบาย: บรรทัดนี้เปรียบเหมือนการติดตั้งเสาส่งวิทยุ โดยระบุว่าท่อนี้จะรับส่งเฉพาะข้อมูลแบบ Candle และมีพื้นที่พักข้อมูล (Buffer) 1,024 ชุด เพื่อป้องกันข้อมูลค้าง 2. การสร้างตัวรับสัญญาณใหม่ (Subscription)คำสั่ง: let mut rx = state.tx.subscribe();ใช้เพื่อ: สร้าง "เครื่องรับวิทยุ" ให้กับ Browser แต่ละรายที่เชื่อมต่อเข้ามาอธิบาย: เมื่อมี User คนใหม่เปิดหน้าเว็บ (WebSocket เชื่อมต่อสำเร็จ) Rust จะใช้คำสั่งนี้เพื่อสร้าง rx (Receiver) ประจำตัวของ User คนนั้น ทำให้เขาสามารถฟังข้อมูลราคาที่ tx กำลังประกาศอยู่ได้ 3. การประกาศราคาออกอากาศ (Broadcasting Data)คำสั่ง: let _ = tx.send(candle);ใช้เพื่อ: ส่งข้อมูลราคาที่ได้รับจาก Deriv เข้าไปในท่อกลางอธิบาย: เมื่อ Task ที่เชื่อมต่อกับ Deriv ได้ราคาใหม่มา (ผ่านฟังก์ชัน connect_to_deriv) เขาจะใช้คำสั่งนี้ "ตะโกน" ข้อมูลราคาลงใน tx ซึ่งจะส่งผลให้ rx ของผู้ใช้ทุกคนได้รับราคาพร้อมกันทันที4. การดึงข้อมูลมาใช้งาน (Receiving Data)คำสั่ง: while let Ok(candle) = rx.recv().awaitใช้เพื่อ: รอฟังและดึงข้อมูลราคาออกจากเครื่องรับ (rx) เพื่อส่งออกไปทาง WebSocketอธิบาย: คำสั่งนี้จะ "รอ" (Await) จนกว่าจะมีข้อมูลส่งมาจาก tx เมื่อได้ข้อมูลมาแล้วก็นำไปแปลงเป็น JSON และส่งไปหาหน้าจอ Browser ต่อไป 📊 สรุปความสัมพันธ์ในรูปแบบ Diagramคำสั่งใครเป็นคนใช้?หน้าที่หลักtx.subscribe()Browser Taskเปิดเครื่องรับเพื่อรอฟังราคาtx.send(data)Deriv Taskส่งราคาใหม่ลงในระบบเพื่อกระจายเสียงrx.recv()Browser Taskหยิบราคาจากเครื่องรับมาแสดงผล
